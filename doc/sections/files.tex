\section{ファイル構成}

\subsection{標準ディレクトリ構造}

本アーキテクチャに準拠するプロジェクトは、以下のディレクトリ構造に従う。
本リポジトリでは \texttt{sample/} および \texttt{verified/} がそれぞれこの構造を持つ独立したPlatformIOプロジェクトである。

\begin{lstlisting}[basicstyle=\ttfamily\small, frame=single]
ProjectRoot/
+-- include/                    ... プロジェクト固有ヘッダー
|   +-- SystemData.h            ... SystemData構造体（モジュール間共有データ）
|   +-- ProjectConfig.h         ... Configインスタンス定義（ピンアサイン等）
+-- lib/
|   +-- ModuleCore/             ... コアライブラリ（プロジェクト非依存）
|   |   +-- IModule.h           ... IModuleインターフェース
|   |   +-- ModuleTimer.h       ... ModuleTimerクラス
|   +-- {ModuleName}/           ... 各モジュール実装（1モジュール1ディレクトリ）
|       +-- {ModuleName}.h      ... Config/Data構造体 + クラス宣言
|       +-- {ModuleName}.cpp    ... メソッド実装
+-- src/
|   +-- main.cpp                ... エントリーポイント（モジュール登録・ループ制御）
+-- platformio.ini              ... PlatformIO設定（-I include を含む）
\end{lstlisting}

\subsection{ファイル役割一覧}

\begin{tabularx}{\textwidth}{l l X}
\toprule
\textbf{ファイル} & \textbf{種別} & \textbf{説明} \\
\midrule
\texttt{lib/ModuleCore/IModule.h} & コア & IModuleインターフェース定義（プロジェクト非依存） \\

\subsection{ライブラリ構成方針}

\begin{itemize}
    \item \textbf{lib/ModuleCore/} — プロジェクト非依存のコアコンポーネント（IModule、ModuleTimer）。
    他のプロジェクトにもそのまま流用可能。
    \item \textbf{lib/\{ModuleName\}/} — 個別のハードウェアモジュール実装。
    それぞれが\texttt{IModule}を継承し、モジュール固有のConfig/Data構造体を定義する。
    \texttt{.h}（宣言 + 前方宣言）と\texttt{.cpp}（実装）に分離する。
    1モジュール1ディレクトリの原則に従う。
    \item \textbf{include/} — プロジェクト固有のヘッダー。
    \texttt{SystemData.h}は使用するモジュールのData構造体を集約し、
    \texttt{ProjectConfig.h}はConfigインスタンス（ピン番号等の具体的な値）を定義する。
\end{itemize}
